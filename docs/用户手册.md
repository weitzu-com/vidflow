## 一、核心理念：短视频的第一性原理

### 1.1 短视频的物理学本质

从第一性原理出发，一条短视频只有三个不可再分的物理要素：

```
短视频 = 画面流(Video Stream) × 声音流(Audio Stream) 沿着 时间轴(Timeline) 并行展开
```

| 要素 | 物理形式 | 数字表示 | 创作方式 |
|------|---------|---------|---------|
| 画面 | 1920×1080 像素矩阵 × 25fps | H.264 编码的视频轨 | Seedance AI 生成 / 实拍 / HTML 动画 |
| 声音 | 44100Hz 采样率 PCM 音频 | AAC 编码的音频轨 | SAMI TTS / edge-tts / macOS say |
| 时间轴 | 微秒级时间戳序列 | FFmpeg concat + SRT 字幕 | 音频实测时长驱动 |

**核心洞察**：只要掌控这三个要素，剪辑软件只是"胶水"。真正的创造性工作在于剧本、提示词、声音设计。

### 1.2 为什么这个手册存在

在制作「安史之乱」短视频的过程中，我们踩了 29 个坑，迭代了 5 个版本，从最初 4 分的 HTML 动画进化到 8.5 分的暗黑史诗大师版。这个过程暴露了一个核心问题：

> **AI 视频创作不缺工具，缺的是把所有工具串联起来的操作手册。**

本手册填补这个空白。

### 1.3 五版进化一览

| 版本 | 时长 | 画面方案 | 音频方案 | 核心坑 | 评分 |
|------|------|---------|---------|--------|------|
| v1 | 115s | HTML/CSS 动画 | 剪映纪录片男声 | 剪映 10.7 不兼容 5.9 格式 | 4/10 |
| v2 | 115s | Seedance AI 8/10 | 同上 | 音画不同步 + 审核拦截 | 6/10 |
| v3 | 228s | Seedance AI 8/10 | macOS Tingting | 文案过长 + 语音不清 | 5/10 |
| v4 | 94s | Seedance AI 10段 | 激昂旁白 | 视频定格 + 语速偏快 | 7/10 |
| **v5** | **73s** | **暗黑 AI 5段** | **沉稳解说 BV701** | **末尾定格 16s** | **8.5/10** |

## 四、阶段一：剧本创作

### 4.1 Notebook LM 工作流（本项目实际采用）

**背景**：v1-v4 版本的剧本由 Claude 直接生成，质量中规中矩。v5 大师版的剧本改用 Notebook LM 先整理历史素材，再按结构化提示词输出分镜脚本。

**完整流程**：

```
第1步: 在 Notebook LM 中上传安史之乱相关史料
第2步: 让 Notebook LM 自动整理关键事件时间线、人物关系、数据
第3步: 输入结构化提示词 → Notebook LM 输出分镜头脚本
第4步: 将脚本交给 Claude → 拆解为 Seedance 提示词 + TTS 文案段
```

**实际使用的 Notebook LM 提示词**：

```
我需要制作一个2分钟的竖屏短视频，主题是安史之乱。
请根据你掌握的资料，为我写一份 450 字左右的短视频分镜头脚本。

必须遵循以下结构：

黄金前 3 秒 (Hook)：
打破常规认知，提出一个悬念
（例如：大唐的毁灭，真的是因为一个女人吗？）。

第 3-45 秒 (底层逻辑)：
用通俗的语言解释"外重内轻"的根本原因（第一性原理）。

第 45-90 秒 (高潮画面)：
描述渔阳鼙鼓动地来和马嵬驿之变的视觉画面，节奏要快。

第 90-120 秒 (余波与反思)：
用震撼的数据对比结尾，升华主题。
```

**⚠️ 坑点7**: Notebook LM 输出的剧本是"散文式"描述，不适合直接用于 Seedance 提示词。需要 Claude 二次加工：将画面描述翻译为英文提示词，将旁白拆分为 TTS 句子段。

## 五、阶段二：AI 视频生成

### 5.1 Seedance API 完整调用

```python
import requests, time

API_KEY = "your-api-key"  # 火山引擎 ARK API Key
API_BASE = "https://ark.cn-beijing.volces.com/api/v3"
MODEL_ID = "doubao-seedance-2-0-260128"
HEADERS = {"Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json"}

def seedance_generate(prompt, duration=8, resolution="720p"):
    """
    完整流程: 提交任务 → 轮询等待 → 下载视频
    返回: 本地文件路径 或 None
    """

    # ── 第1步: 提交任务 ──
    body = {
        "model": MODEL_ID,
        "content": [{"type": "text", "text": prompt}],
        "ratio": "16:9",
        "resolution": resolution,
        "duration": duration,
        "generate_audio": False,   # 静音！旁白单独合成
        "return_last_frame": False,
    }
    r = requests.post(
        f"{API_BASE}/contents/generations/tasks",
        headers=HEADERS, json=body, timeout=30
    )

    # ⚠️ 坑点8: 不检查 HTTP 状态码直接 .json() → 异常崩溃
    if r.status_code != 200:
        raise Exception(f"提交失败 HTTP {r.status_code}: {r.text[:300]}")

    task_id = r.json()["id"]
    print(f"任务已提交: {task_id}")

    # ── 第2步: 轮询等待 ──
    # ⚠️ 坑点9: 任务可能排队很久（实测 30s-519s），必须设置合理超时
    start = time.time()
    while time.time() - start < 600:  # 10 分钟超时
        r = requests.get(
            f"{API_BASE}/contents/generations/tasks/{task_id}",
            headers=HEADERS, timeout=10
        )
        # ⚠️ 坑点10: 轮询时 HTTP 可能失败（网络波动），不要直接抛异常
        if r.status_code != 200:
            time.sleep(5)
            continue

        data = r.json()
        status = data.get("status")

        if status == "succeeded":
            video_url = data["content"]["video_url"]
            break
        elif status == "failed":
            error = data.get("error", {}).get("message", "未知错误")
            raise Exception(f"生成失败: {error}")
        elif status in ("queued", "running"):
            time.sleep(10)  # 退避等待
        else:
            time.sleep(5)

    else:
        raise TimeoutError(f"任务 {task_id} 在 600s 内未完成")

    # ── 第3步: 下载视频 ──
    # ⚠️ 坑点11: video_url 仅 24 小时有效！
    # ⚠️ 坑点12: 下载可能因网络超时失败，需要重试
    for attempt in range(3):
        try:
            r = requests.get(video_url, timeout=120)
            if r.status_code == 200:
                return r.content  # 或保存到文件
        except Exception as e:
            print(f"下载重试 {attempt+1}/3: {e}")
            time.sleep(5)
    return None
```

### 5.2 并行生成策略

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

# ⚠️ 坑点13: 串行生成太慢（10段×2分钟=20分钟）
# 解决: 三阶段并行 — 提交/轮询/下载各自并行

def generate_all_parallel(scenes, max_workers=10):
    task_ids = {}

    # 阶段A: 并行提交（所有任务同时发送，1s 内完成）
    with ThreadPoolExecutor(max_workers=max_workers) as ex:
        futures = {
            ex.submit(seedance_generate, s["prompt"], s["duration"]): s["name"]
            for s in scenes
        }
        for f in as_completed(futures):
            name = futures[f]
            try:
                task_ids[name] = f.result()  # 阻塞直到单个任务完成
            except Exception as e:
                print(f"  ❌ {name}: {e}")

    return task_ids
```

### 5.3 暗黑史诗风格提示词配方

经过大量 A/B 测试，以下是已验证有效的暗黑电影级提示词公式：

```
基础模板（历史场景）:
"[主体描述], dark cinematic lighting, deep shadows,
 moody atmosphere, atmospheric haze, high contrast,
 desaturated colors with deep blacks, film grain texture,
 cinematic wide shot, epic historical documentary aesthetic"

温度调节:
  更暗: + "minimal lighting, almost black shadows, single dim light source"
  更冷: + "cool blue undertones, steel grey palette, moonlit only"
  废墟: + "silhouettes against dying sunset, ash particles in last light"
```

### 5.4 Seedance 审核规避指南

| 审核类型 | 触发词（被拒） | 安全替换 |
|------|------|------|
| 敏感信息 | `military markers spreading` | `trade route markers expanding` |
| 暴力内容 | `blood-red sunset` | `crimson sunset` (⚠️ 含 blood 会被拦截) |
| 人物肖像 | `emperor on throne` | `empty throne in palace` |
| 版权内容 | `Disney style` | `epic cinematic style` |

**⚠️ 坑点14**: `blood-red` 这个常用英文短语会触发审核！解决方案：用 `crimson`、`deep amber`、`dying sunset` 替代。

### 5.5 常见 Seedance 错误码

| HTTP 状态 | 错误类型 | 原因 | 解决方案 |
|------|------|------|------|
| 200 + status=failed | 内容审核拦截 | 提示词含敏感词 | 替换触发词（见上表） |
| 200 + status=expired | 任务过期 | 超过 `execution_expires_after` | 重新提交 |
| 401 | AuthenticationError | API Key 错误或过期 | 检查 Key |
| 404 | InvalidEndpointOrModel | 模型 ID 错误 | doubao-* vs dreamina-* 区分 |
| 403 | Forbidden (下载) | video_url 超过 24h | 任务成功后立即下载 |

### 6.3 从草稿提取精确时间轴

```python
def extract_subtitles(draft_path):
    """从 draft_content.json 提取字幕，实测时长精确到微秒"""
    with open(os.path.join(draft_path, "draft_content.json")) as f:
        draft = json.load(f)

    # 映射 material_id → 文字
    text_map = {}
    for mt in draft["materials"]["texts"]:
        c = json.loads(mt["content"])
        text_map[mt["id"]] = c.get("text", "")

    # 提取字幕段
    subtitles = []
    for track in draft["tracks"]:
        if track["type"] == "text":
            for seg in track["segments"]:
                mid = seg.get("material_id", "")
                tr = seg.get("target_timerange", {})
                start = tr.get("start", 0) / 1e6  # 微秒→秒
                dur = tr.get("duration", 0) / 1e6
                text = text_map.get(mid, "")
                if text and dur > 0:
                    subtitles.append((start, start + dur, text))

    subtitles.sort()

    # ⚠️ 坑点17: 标点符号被拆分为独立 segment
    # "你好。" → [("你好", 0-1s), ("。", 1-1.1s)]
    # 解决: 合并短片段
    merged = []
    for s, e, t in subtitles:
        if merged and s - merged[-1][1] < 0.12 and len(merged[-1][2]) < 2:
            ps, _, pt = merged.pop()
            merged.append((ps, e, pt + t))
        else:
            merged.append((s, e, t))

    # ⚠️ 坑点18: 合并阈值不能太大
    # 0.15s 阈值会吞掉句子间的自然停顿 → 字幕变成一大段
    # 0.12s 且仅合并极短片段（<2字）→ 只合并标点符号

    return merged
```

## 七、阶段四：FFmpeg 合成与同步

### 7.1 完整五步合成流水线

```
步骤1: concat OGG → 完整旁白音频 (lossless copy)
步骤2: concat MP4 → 完整画面轨 (lossless copy)
步骤3: 计算填充量 → pad = audio_dur - video_dur
步骤4: 第一遍编码 → tpad延长末帧 + amix混BGM + eq暗黑调色
步骤5: 第二遍编码 → 烧录 SRT 字幕
```

### 7.2 完整代码

```python
def assemble_master(video_paths, ogg_files, subtitles, output_path,
                    bgm_path=None):
    """大师级 FFmpeg 合成"""
    FFMPEG = "/opt/homebrew/opt/ffmpeg-full/bin/ffmpeg"
    FFPROBE = "/opt/homebrew/opt/ffmpeg-full/bin/ffprobe"

    # ── 1. 拼接音频 ──
    alist = "_audio_list.txt"
    with open(alist, "w") as f:
        for af in ogg_files:
            dur = float(subprocess.run(
                f'{FFPROBE} -v error -show_entries format=duration '
                f'-of default=noprint_wrappers=1:nokey=1 "{af}"',
                shell=True, capture_output=True, text=True
            ).stdout.strip())
            f.write(f"file '{af}'\nduration {dur:.3f}\n")

    full_audio = "_full_audio.ogg"
    subprocess.run(
        f'{FFMPEG} -y -f concat -safe 0 -i "{alist}" -c copy "{full_audio}"',
        shell=True, capture_output=True
    )

    # ── 2. 拼接视频 ──
    vlist = "_video_list.txt"
    with open(vlist, "w") as f:
        for vp in video_paths:
            f.write(f"file '{vp}'\n")
    merged_v = "_merged.mp4"
    subprocess.run(
        f'{FFMPEG} -y -f concat -safe 0 -i "{vlist}" -c copy "{merged_v}"',
        shell=True, capture_output=True
    )

    # ── 3. 计算填充 ──
    vdur = float(subprocess.run(
        f'{FFPROBE} -v error -show_entries format=duration '
        f'-of default=noprint_wrappers=1:nokey=1 "{merged_v}"',
        shell=True, capture_output=True, text=True
    ).stdout.strip())
    adur = float(subprocess.run(
        f'{FFPROBE} -v error -show_entries format=duration '
        f'-of default=noprint_wrappers=1:nokey=1 "{full_audio}"',
        shell=True, capture_output=True, text=True
    ).stdout.strip())
    total = max(vdur, adur)
    pad = max(0, total - vdur)

    # ⚠️ 坑点20: 如果视频总长 > 音频总长，音频不够
    # → pad=0，用 -shortest 切齐视频
    # 如果视频总长 < 音频总长，视频不够
    # → pad>0，用 tpad 延长末帧

    # ── 4. 第一遍编码 ──
    temp = "_temp.mp4"
    has_bgm = bgm_path and os.path.exists(bgm_path)

    # ⚠️ 坑点21: -vf 和 -filter_complex 不能同时用于同一视频流
    # 解决: 把 eq 调色放入 filter_complex，字幕用单独的第二遍编码
    # ⚠️ 坑点22: filter_complex 中的逗号是 filter 分隔符
    # 解决: force_style 的逗号用 \, 转义，或者分两次编码

    if has_bgm:
        cmd = (
            f'{FFMPEG} -y '
            f'-i "{merged_v}" -i "{full_audio}" -i "{bgm_path}" '
            f'-filter_complex "'
            f'[0:v]tpad=stop_mode=clone:stop_duration={pad:.1f},'
            f'eq=brightness=-0.06:contrast=1.15:saturation=0.75[v];'  # ← 暗黑调色
            f'[2:a]atrim=0:{total:.1f},volume=0.10[bgm];'
            f'[1:a][bgm]amix=inputs=2:duration=first[a]'
            f'" '
            f'-map "[v]" -map "[a]" '
            f'-c:v libx264 -preset slow -crf 18 '
            f'-c:a aac -b:a 192k -pix_fmt yuv420p '
            f'"{temp}"'
        )
    else:
        cmd = (
            f'{FFMPEG} -y '
            f'-i "{merged_v}" -i "{full_audio}" '
            f'-filter_complex "'
            f'[0:v]tpad=stop_mode=clone:stop_duration={pad:.1f},'
            f'eq=brightness=-0.06:contrast=1.15:saturation=0.75[v]'
            f'" '
            f'-map "[v]" -map 1:a '
            f'-c:v libx264 -preset slow -crf 18 '
            f'-c:a aac -b:a 192k -pix_fmt yuv420p '
            f'"{temp}"'
        )
    subprocess.run(cmd, shell=True, capture_output=True)

    # ── 5. 第二遍编码（烧录字幕）──
    # ⚠️ 坑点23: 字幕滤镜需要 libass (只有 ffmpeg-full 有)
    # ⚠️ 坑点24: subtitles 滤镜的路径不能含特殊字符

    srt_path = "_subtitles.srt"
    with open(srt_path, "w", encoding="utf-8") as f:
        for i, (s, e, t) in enumerate(subtitles, 1):
            h = lambda x: f"{int(x//3600):02d}:{int((x%3600)//60):02d}:{x%60:06.3f}".replace(".", ",")
            f.write(f"{i}\n{h(s)} --> {h(e)}\n{t}\n\n")

    subprocess.run(
        f'{FFMPEG} -y -i "{temp}" '
        f'-vf subtitles="{srt_path}" '
        f'-c:v libx264 -preset slow -crf 18 -c:a copy '
        f'"{output_path}"',
        shell=True, capture_output=True
    )

    # ── 清理 ──
    for f in [alist, vlist, full_audio, merged_v, temp, srt_path]:
        if os.path.exists(f): os.remove(f)

    return output_path
```

